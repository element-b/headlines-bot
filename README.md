# Headlines Bot

Асинхронный Telegram-бот «Агрегатор новостей» для сбора, хранения и доставки новостных заголовков из нескольких источников.

Бот поддерживает два режима работы:

- получение новостей по запросу;
- автоматическую отправку новых заголовков по подписке.

Проект разработан как тестовое задание на позицию Python-разработчика.

---

## Возможности

- сбор заголовков из 4 источников:
  - The Guardian — через API
  - The New York Times — через API
  - Коммерсантъ — через RSS
  - РБК — через RSS
- хранение заголовков, пользователей и подписок в PostgreSQL
- выдача последних новостей по команде
- подписки на источники
- автоматическая рассылка новых заголовков подписчикам
- дедупликация по URL
- настройка источника по умолчанию для команды `/news`
- настройка количества заголовков в выдаче
- раздельные фоновые сервисы:
  - `Scraper`
  - `Notifier`
  - `Bot`

---

## Команды бота

- `/start` — запуск бота и краткое описание
- `/help` — список доступных команд
- `/faq` — как работает бот
- `/news` — последние новости из источника по умолчанию
- `/news_all` — последние новости из всех источников
- `/settings` — выбор источника по умолчанию и количества заголовков
- `/subscribe` — подписка на источники
- `/unsubscribe` — отмена подписки
- `/mysubs` — просмотр активных подписок

## Как работает бот

Бот работает в двух режимах.

### 1. Получение новостей по запросу
Пользователь сам запрашивает последние новости:

- `/news` — из источника по умолчанию;
- `/news_all` — из всех доступных источников.

### 2. Получение новостей по подписке
Пользователь подписывается на один или несколько источников и получает автоматические уведомления только о новых заголовках, появившихся после подписки.

---

## Важное различие: источник по умолчанию и подписка

### Источник по умолчанию
Источник по умолчанию используется командой `/news`.

Пример:
- пользователь выбрал `The Guardian` как источник по умолчанию;
- при вызове `/news` бот покажет последние новости именно из `The Guardian`.

### Подписка
Подписка используется для автоматических уведомлений.

Пример:
- пользователь подписан на `РБК` и `The New York Times`;
- бот будет автоматически отправлять новые заголовки из этих источников.

### Почему это разные сущности
Они решают разные задачи:

- источник по умолчанию — для ручного запроса;
- подписки — для автоматической доставки.

Подписка на источник **не делает его автоматически источником по умолчанию**.

## Источники данных

### The Guardian
- тип: `api`
- сайт: `https://www.theguardian.com`
- API: `https://content.guardianapis.com/search`

### The New York Times
- тип: `api`
- сайт: `https://www.nytimes.com`
- API: `https://api.nytimes.com/svc/search/v2/articlesearch.json`

### Коммерсантъ
- тип: `rss`
- RSS: `https://www.kommersant.ru/RSS/news.xml`

### РБК
- тип: `rss`
- RSS: `https://rssexport.rbc.ru/rbcnews/news/30/full.rss`

## Архитектура

Проект разделён на несколько независимых частей.

### `Bot`
Отвечает за:
- Telegram polling;
- обработку команд;
- выдачу новостей из базы данных;
- настройки пользователя;
- управление подписками.

### `ScraperService`
Отвечает за:
- периодический обход активных источников;
- получение новых заголовков;
- сохранение новостей в базу данных.

### `NotifierService`
Отвечает за:
- выборку ещё не отправленных пользователю заголовков;
- автоматическую отправку новостей подписчикам;
- фиксацию уже отправленных новостей в БД.

### `PostgreSQL`
Используется для хранения:
- источников;
- заголовков;
- пользователей;
- подписок;
- истории уже отправленных уведомлений.
